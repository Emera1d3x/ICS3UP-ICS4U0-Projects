In Computer Science, recursion is a very powerful tool. Recursion is the process in which a method/function calls itself, thus creating a loop. This loop can be very useful, it can do a repetitive task with little code, and breakdown a complicated problem in to smaller managable steps.

Here is some code of a method calling itself, which is recursion:


        (Recursive Counter)

    1     public void counter (int variablePassed){
    2         System.out.println(variablePassed);
    3         counter(variablePassed+1); // Calling itself, thus creating a loop
    4     }

In Java, a recursive call is put on a "stack". Say you are calling a method within itself, this adds a method call on top of the stack. You can think of the stack data-structure as a stack of books. If you put too much books on top of eachother, eventuallly you'll build too high and hit the ceiling, thus leading to the error of having too much information and running out of space. In Java this is known as a Stack Overflow Error (the bane of my existence) - when there are way too many calls (this usually happens when you have a logic error). In order to stop the infinite method calls, you must have a base case. A base case is a condition within the recursive loop, which stops the recursive calls from continuing.

Here is the code but with a base case stopping the printing when the integer being passed through is 100.


        (Recursive Counter with Base Case)

    1     public void counter (int variablePassed){
    2         System.out.println(variablePassed);
    3         if (variablePassed!=100) // Base case stopping infinite loop
    4             counter(variablePassed+1);
    5     }

Recursion Implementation: Collatz Conjecture

For the recursion example I decided to make an implementation of a math problem called Collatz Conjecture.
(I didn't really want a factorial or fibonacci sequence calculator, that has probably already been done millions of time before)
I got inspired by this cool video I watched a few years ago: YoutubeLink

This is how the formula works. Given a number, if it is even, divide it by two, if it is odd, multiply it by 3 and add 1. What makes this problem so unique is that when you track the "life" of a number as it continiously iterates, its "adventures" are very interesting and seem random. With all numbers, it eventually "settles" down and reaches 1 (my program eliminates the base loop to avoid stack overflow). If you use any natural numbers and cause a stackoverflow (assuming Java is perfect), you found the first ever cycle (apart from the base loop). This program goes through the lifespan of user input and outputs the peak number it reaches:


        (Finding Maximum Collatz Conjecture)

    1     public int findMaxCollatzConjecture(int n, int max){
    2         System.out.println(n);
    3         if (n==1) // Base Case (Prevents Infinite Loop)
    4             return max;
    5         if (n%2==1)
    6             return findMaxCollatzConjecture((3*n+1), Math.max(max,n)); // Recursive Call (Returns next term's answer)
    7         else
    8             return findMaxCollatzConjecture((n/2), Math.max(max,n)); // Recursive Call (Returns the next term's answer)
    9     }
   10
   11     public static void main(String[] args){
   12         Main z = new Main();
   13         Scanner s = new Scanner(System.in);
   14         int check = s.nextInt();
   15         System.out.println("The max is: "+z.findMaxCollatzConjecture(check, 1));
   16     }
